BACKGROUND AND RATIONALE - Part 1

Bioinformatics workflows may not be optimized for performance

This research will investigate two contributing factors affecting performance

  1. R and Python computing acceleration using optimal linear algebra libraries (BLAS/LAPACK) such as those provided by the reference implementation, openblas, ATLAS, Apple Accelerate (ARM), AMD BLISS, Intel MKL and usage of other computing frameworks such as OpenCL
  1. Software patches deployed as mitigations to Meltdown (CVE-2017-5754) and Spectre (CVE-2017-5753 and CVE-2017-5715) vulnerabilities which are known to impact processor computing performance

Acceleration by linear algebra libraries can significantly reduce computational time in R

Acceleration by linear algebra libraries can significantly reduce computational time in Python

There has been documented performance regressions when running MAFFT Alignment benchmark test as part of the Phoronix Test Suite with Spectre/Metltdown Mitigations

There are other performance regressions noted with other tests of the Phoronix Test Suite with Spectre/Metltdown Mitigations

BACKGROUND AND RATIONALE - Part 2

Replicate an unsupervised machine learning framework for discriminating major variants of concern during COVID-19 and apply to PH data in GISAID

Visualization 1: Discriminate the COVID19 major variants of concern from k-mer data using Principal Components Analysis, Uniform Manifold Approximation and Projection (UMAP) and t-Stochastic Network Embedding (t-SNE)

Visualization 2: Cluster by Dendograms the major variants from k-mer data using agglomerative nesting (AGNES)
